# 一、收益与运营

### 一、制定北极星指标

```text
所有的行动都是有收益的
以开发组件库为例
我们的开发都是为了提效，提效可以分为两部分，接入组件成本更低，组件升级效率更高（如：UI风格统一升级，功能拓展）。当然组件开发也是有成本的，比如组件的开发成本考虑的业务场景和使用场景更多组件开发更复杂，第二点接入是有学习成本的
我们的北极星指标就属于：净收益
```

### 二、北极星指标收益计算

```text
计算公式：净收益 = (开发成本 - 接入成本) * 使用次数
开发成本：组件开发成本，文档撰写，组件库运营（宣传推广），bug修复，组件升级
接入成本：文档阅读，组件接入及测试，如果组件能力不匹配还有一定的组件功能拓展成本（影响体验，须快速支持）
使用次数：每个组件的使用次数，如果一个项目多次使用同一个组件仅算一次
说明：每个组件的开发成本不同，在使用频次越高提效越明显；也有业务组件较为复杂，但是接入方仅使用其一个场景，可能会产生高额成本（接入复杂，组件体积庞大），组件开发需要优先考虑使用体验
```


### 三、北极星指标细分

```text
北极星指标一定是由一些小指标来达成的，我们在指定北极星指标后需要分析具体的小指标有哪些，这样才能真正清晰迅速的提升北极星指标
依然以组件库为例
开发成本：我们在组件库开发提供自动化开发模板生成(代码文件与文档文件)，完善的测试能力，自动化组件发布流程
接入成本：体验优秀的demo页面，提供二维码可在任何环境浏览体验，完善的组件参数说明文档，提供复制粘贴代码的能力，接入简单快捷
使用次数：通过组件统计数据分析了解当前组件的接入量，以及目前项目的数量，计算当前组件对项目的覆盖度和组件的使用率，分析哪些组件使用低频，以了解原因（体验，功能不符合），以及哪些项目未覆盖到，比如宣传不到位或者文档确实或体验不足
```

### 四、北极星指标痛点分析

```text
完成以上细分指标时，我们会逐步体会到其中一些问题
依然以组件库为例
业务对组件表现拒绝，不信任感 有两点原因：KPI项目，可能随时停止维护，功能性不满足时组件研发支持不及时，影响项目进度，缺少专业客服，问题响应慢，担心组件变化太快，组件库更新进度跟不上，再者就是业务方接入组件库并不能对用户有KPI收益，对接入态度积极性不高
问题解决：组件库需要良好的运营起来，组件库在公司级别分享大会，组件库体系技术博客，公司大群推广定期发送重大更新；组件库的官方群并保持官方活跃性，积极推送组件库的最新进展，积极响应用户提出的所有问题，当然有条件可以在群里有几个托经常活跃，官方客服群不活跃很容易被抛弃，一定要积极运营起来才能被用户所信任；积极在所有公司相关技术网站推广组件库的官网入口，提升知名度；组件库可展示所有组件贡献者名单，维护更新时间，提升业务方来贡献组件的活跃度，有条件可以对贡献者进行一些奖励（比如键盘、玩偶）
```

### 五、北极星指标阶段性分析

```text
建立北极星指标我们需要观测每个月统计数据
依然以组件库为例
对每个组件安装或项目发布时我们需要统计上报组件的使用量，组件覆盖率，项目覆盖率，分析数据增长情况，预估每个细分指标的效果和次月统计结果，对比数据分析问题，逐步提升北极星指标数据。每项数据的分析和执行需要做到定位到具体原因，快速上线解决方案，及时广播，观测收益。
```
